# SETUP — Deploying the Blueprint README

This is the deployment guide for the upgraded `GclinTrimble/GclinTrimble` profile README. Everything in this folder is designed to drop into your existing repo. Follow these steps in order.

> **TL;DR** — copy 4 files into the repo, add 1 secret, enable 2 workflows, hit run.

---

## 1. Files to commit

From this folder, copy the following into your `GclinTrimble/GclinTrimble` repository, preserving the directory structure:

```
README.md                                 ← replaces the current README
assets/blueprint-header.svg               ← new header (keep your old terminal-header.svg if you want to revert)
.github/workflows/snake.yml               ← snake animation generator
.github/workflows/metrics.yml             ← metrics dashboard generator
```

Then commit and push to `main`. The README will render immediately on your profile.

```bash
git clone https://github.com/GclinTrimble/GclinTrimble.git
cd GclinTrimble
# (copy the files in)
git add README.md assets/blueprint-header.svg .github/workflows/
git commit -m "feat: blueprint-themed README with federated digital twin storytelling"
git push origin main
```

## 2. Enable the Snake animation

The snake workflow generates a `.svg` of a snake eating your contribution grid, then pushes it to an `output` branch. The README references that branch.

**Steps:**

1. After pushing the files above, go to **Actions** → **Generate Snake Animation** → **Run workflow**.
2. Wait ~30 seconds. It will create an `output` branch with two SVGs.
3. The README image at `https://raw.githubusercontent.com/GclinTrimble/GclinTrimble/output/github-contribution-grid-snake-dark.svg` will now resolve.

If the workflow fails with a permissions error, check **Settings → Actions → General → Workflow permissions** and set it to **Read and write permissions**.

---

## 3. Enable the Metrics dashboard

The metrics workflow uses [`lowlighter/metrics`](https://github.com/lowlighter/metrics) to build a full dashboard SVG in `assets/metrics.svg`.

**Steps:**

1. Create a **Personal Access Token** at https://github.com/settings/tokens.
   - **Classic token works fine.** Scopes needed: `public_repo`, `read:user`, `read:org` (optional, only if you want org stats).
   - Set the expiration to whatever you're comfortable with.
2. Add it as a repository secret:
   - **Settings → Secrets and variables → Actions → New repository secret**
   - Name: `METRICS_TOKEN`
   - Value: paste your PAT
3. Run the **Generate Metrics Dashboard** workflow manually once. It will commit `assets/metrics.svg` back to the repo.

> **Note:** the metrics workflow does write back to the repo (it commits the SVG). That's intentional and expected.

## 4. Enable the Spotify now-playing widget

The `§ 07 — NOW PLAYING` section uses [`kittinan/spotify-github-profile`](https://github.com/kittinan/spotify-github-profile), which is a free hosted service you authenticate once via Spotify OAuth.

**Steps:**

1. Visit https://spotify-github-profile.kittinanx.com/
2. Click **Login with Spotify** and authorize the app.
3. After the redirect, the page will give you a personalized image URL containing your Spotify UID.
4. Open `README.md` and replace the `uid=GclinTrimble` parameter in the Spotify `<img>` `src` with your real UID. Everything else (theme, colors, `interchange`) is already tuned for the blueprint palette.

If you don't want to set this up, just delete `§ 07 — NOW PLAYING` from the README — nothing else depends on it.

## 5. Fill in the placeholders

The README is intentionally truthful about being a template in a couple of spots. Edit these to taste:

- **`§ 05 — DELIVERABLES & WINS`** — swap `[N]`, `[days]`, `[hours]`, etc. with real numbers. If you're not comfortable putting public numbers, replace with qualitative wins ("shipped the v1 federated layer").
- **`§ 06 — CURRENT JOB QUEUE`** — keep or replace with whatever you're actually working on.
- **Spotify UID** — see step 4.

## 6. Reverting

If you want to roll back to the old matrix-green terminal README:

- The old README is still in your repo's git history (`git log README.md`).
- The old header SVG (`assets/terminal-header.svg`) is still on `main` unless you explicitly deleted it.
- `git revert <commit>` on the README change brings everything back.

## Troubleshooting

**Images show as broken on the profile.**
The README references images on the `output` branch (snake) and `main` branch (metrics, header). If either workflow hasn't completed yet, those images won't resolve. Run the workflows manually once and they'll start rendering.

**The badges look wrong / off-color.**
Shields.io occasionally caches old badge renders. Hard-refresh (`Cmd+Shift+R`) or wait ~10 minutes.

**Stats card says "Could not fetch user".**
The [`github-readme-stats`](https://github.com/anuraghazra/github-readme-stats) service is rate-limited. It usually self-heals within an hour. If it persists, switch the URL host to `github-readme-stats-eight-theta.vercel.app` (community mirror).

**The streak stats card 404s.**
The Heroku host for `github-readme-streak-stats` has had outages. The current canonical URL is `https://github-readme-streak-stats.herokuapp.com/` — if it fails, swap to `https://streak-stats.demolab.com/` (same params, different host).
